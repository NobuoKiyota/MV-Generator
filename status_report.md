## 📌 1. 実装された成果物サマリー

1. **[xlsx_generator_gui.py](file:///z:/mv-generator/xlsx_generator_gui.py)** (2タブ版決定稿)
   - `ttk.Notebook` を用いた「設定制作（タブ1）➔ タイムライン展開（タブ2）」の2ステップ・トップダウンワークフローを搭載した軽量なデスクトップGUIアプリ。
   - タブ1の右側に **プレビュー画像表示エリア** および **「テスト画像を生成」** ボタンを配置し、決定稿となった世界観・キャラ設定から1枚プレビュー画像を生成し、視覚的に合意してから展開に進めるプロトタイプ機能を追加。
   - **セッションキャッシュ自動復元機能**: アプリ終了時および生成成功時に、現在のすべての入力フォーム状態、Geminiの生成したテキスト設定、表示中のプレビュー画像パスを `assets/last_session.json` へ自動保存。次回起動時に一瞬で復元され、無駄な API Quota（トークン）の消費を完全にゼロ化。
2. **[utils.py](file:///z:/mv-generator/utils.py)**
   - **MIDI歌詞アライメント同期エンジン (`align_lyrics_with_midi`)**:
     - `mido` ライブラリを使用してMIDIのテンポマップとノートオン（発音イベント）を解析し、歌詞フレーズ（行）とミリ秒単位で完全に自動同期したタイムライン枠（秒数）を計算。
     - Gemini API に対しては「時間計算」を任せず、この確定したタイムライン枠とマスター設定（タイトル、世界観、キャラ、カラー設計）を流し込み、画像プロンプト（英語）と演出説明を充填させるハイブリッド設計。
   - 設定 summary から1枚の高品質な英語画像プロンプトを合成する `generate_test_image_prompt` の実装。
3. **[generate_image_cmd.py](file:///z:/mv-generator/generate_image_cmd.py)**
   - ローカルGPU（CUDA）による Stable Diffusion 描画処理。
   - CUDA が利用できない環境であっても、エラーでクラッシュせず自動的に **CPU 動作（`device="cpu"`, `torch_dtype=torch.float32`）に自動フォールバック** して生成を完遂するセーフティネットを実装。
4. **[verify_gui_scenario.py](file:///z:/mv-generator/verify_gui_scenario.py)**
   - UIの初期配置、インプットの流し込み、STEP 1（設定生成）、プレビュー画像生成、STEP 2（Excel保存と内容チェック）、およびセッションキャッシュの保存・復元までの一連のユーザー操作ライフサイクルを、画面表示なしで全自動シミュレート・検証する自動テストスクリプト。

## 🧠 2. AIによる完全自動化における本質的な限界（反省点）

本開発プロセスを通じ、現在の生成AI技術を用いて「人間が納得するPVを完全自動（1発ボタン）で生成する」ことに対する構造的な限界が浮き彫りになりました。

1. **抽象（音楽）から具象（映像）への翻訳ギャップ**:
   - 音楽や歌詞は極めて抽象的で個人の美的センスに依存するため、AIが一般論（確率論）で連想するイメージと、クリエイターが頭の中で描くピンポイントのこだわり演出との間には必ず大きな乖離が生じる。
2. **物理的時間軸に対するLLMの非互換性**:
   - LLMは本質的にテキスト接続モデルであり、物理的な「時間（ミリ秒）」や「小節ビート（グリッド）」を体感的に把握して正確にアライメントする能力はない。そのため、時間計算をLLMプロンプトに丸投げした初期モデルは5秒以上の大ズレを多発させた。
3. **ビジュアル（キャラクター・画風）の一貫性保持の難しさ**:
   - 画像生成AI（Stable Diffusion等）は1枚1枚独立して推論を行うため、同じプロンプトを使ってもカット間でキャラクターの顔立ちや服装、背景トーンが微妙に変化してしまう。これらを一貫させるには、AI単体での自動化ではなく、LoRA等の個別学習モデルの使用や、人間による手動のマスク手直しが不可欠である。

## 🏁 3. 今後の課題・構想再考へのインサイト

本プロジェクトを将来的に再起動、または別の構想にピボットする際、以下の設計思想が再出発のヒントとなります。

- **AIを「自動運転車」ではなく「パワーステアリング」として位置づける**:
  - 全てを自動で完了させようとすると破綻するため、面倒な作業（MIDIからの時間グリッド切り出し、英語プロンプトの基礎構築）をAIやプログラムに肩代わりさせ、クリエイティブな決定（世界観の決定、キャラクターのディテール、最終テイクの選定）は人間が「調律・ディレクション」するハイブリッド型ツールこそが現実的である。
- **メタパラメータ（制御変数）による映像演出設計**:
  - AIに直接テキストでコンテを書かせるのではなく、ユーザーが「感情の起伏」や「演出密度（BPM、カットの細かさ、アクションの激しさ）」をグラフや数値で指定し、それをプログラムが演出密度に変換した上で、AIに各スロットのプロンプトを生成させるような「メタパラメータ制御」の構想設計が考えられる。
