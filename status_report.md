# 📊 プロジェクト進捗・現状報告書 (Status Report)

本プロジェクトを別PC（自宅のRTX 4060環境や、他PCなど）へ移行し、作業をシームレスに再開するための現状サマリーと未解決の課題リストです。

---

## 1. 現在の進捗状況 (実績)

### 【フェーズ1: タイムライン作成・音響解析】 ➔ 完了
*   **音響解析**: `librosa` を用い、BGM音源（mp3/wav）からBPMと拍（ビート）のタイミングを自動解析。インスト曲にも対応。
*   **タイムライン自動生成**: 歌詞（セクションタグ付き）と演出プロット（優先度別）、および解析BPMから、拍に同期したカット割りをGemini 2.5 Flashで自動設計。
*   **データ確定**: スプレッドシート（data_editor）感覚で手動手直しを行い、確定したデータをJSON/Excelで双方向ロード・セーブ可能。
*   **ZIP一括保存**: タイムラインデータ（JSON）、音源（mp3）、生成されたカット画像をすべて1つの `.zip` パッケージファイルにまとめて一括保存・復元できる機能を実装。

### 【フェーズ2: ローカルGPU画像生成とスライドショー動画化】 ➔ 完了
*   **ローカルGPU画像生成 (Stable Diffusion)**:
    *   PCのグラボ（RTX 3060/4060等のCUDA）を自動検出し、軽量かつ高品質な **`Lykon/dreamshaper-8`** モデルを用いて、完全無料・無制限・爆速（1枚あたり約2〜3秒）で画像をローカル生成するハイブリッドロジックを実装。
    *   有料プランのAPIキーがなくてもローカル完結で絵コンテ画像の生成が可能。
*   **高速Vコンテ動画書き出し (MoviePy)**:
    *   画像をまだ生成していないカットに「歌詞と構成情報が書かれた仮画像（プレースホルダー）」を自動挿入し、音源と連結した仮MV（MP4）を合成。
    *   書き出しフレームレートを `fps=2` に下げ、`ultrafast` プリセットおよび4スレッドのマルチスレッドエンコードを適用し、レンダリング時間を従来の1/10以下（数十秒レベル）に爆速化。

## 2. 動作環境の設定（PC移行時の注意）

### ① 仮想環境 (venv) のパスについて
*   **注意**: Cドライブの空き容量不足（当時残4.4GB）のため、クリーンインストール時に書き込みエラーが発生しました。
*   **対応**: 仮想環境は **`F:\mv-generator-venv`** に構築されています。
*   **移行先PCでの対応**: 移行先PCで `run.bat` を実行する際は、移行先PCの仮想環境フォルダの場所に合わせて **`run.bat` の最終行のパス** を適宜書き換えてください。

### ② 初回起動時のモデル自動ロード
*   **初回のみ**: 「画像を生成」ボタンを初めてクリックした際、Hugging Faceからモデルファイル（約2GBのDreamShaper-8）の自動ダウンロードがバックグラウンドで実行されます。
*   このダウンロード中はStreamlitが数分間「Running...」となって待たされますが、2回目以降はキャッシュされるため2〜3秒で爆速動作します。

## 3. 今後の課題・ネクストステップ (Next Steps)

### 📌 課題1: 初回モデルダウンロードのGUI演出
*   **現状**: 初回画像生成時のダウンロード中、画面上の見た目が「Running...」とだけ表示されフリーズしたように見えてしまう。
*   **改善案**: モデルダウンロードの進行状況（プログレスバー）を表示するか、事前にローカルキャッシュへダウンロードを済ませておくためのウォームアップスクリプトを用意する。

### 📌 課題2: プロンプト改善（日本語リファイン）のプロンプトチューニング
*   **現状**: 日本語で「雨を降らせて」などと入力した際、Gemini 2.5 Flashで英語プロンプトをブラッシュアップする機能はあるが、よりユーザーの狙った構図や音響の雰囲気を細かく反映しやすくするためのシステムプロンプトの調整余地がある。

### 📌 課題3: スライドショー動画の演出エフェクト追加 (候補Bへ)
*   **現状**: 現在のスライドショーはパッパッと画像が切り替わるシンプルな構成（高速化重視）。
*   **改善案**: 「ゆっくりとズームイン・ズームアウトする効果（Ken Burns効果）」や、「カット間のクロスフェード（トランジション）」をオプションとしてチェックボックスで有効化できるようにする。

### 📌 課題4: 本番用動画生成AI（Runway, Luma, Kling 等）への接続
*   **現状**: 現在は静止画デモコンテまで。
*   **改善案**: 確定したカット割りプロンプトとタイミング情報を使い、本番の動画生成AIのAPI（またはエクスポートデータ）と連携して本番用クリップを組み立てるフローの設計。
